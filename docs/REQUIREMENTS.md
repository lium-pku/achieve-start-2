# 时间小达人 · 需求文档

> 面向小学生的家庭时间管理应用，支持孩子/妈妈/爸爸三种角色协同。
> 本文档是产品的**单一事实来源**，所有功能开发与测试用例均以此为准。

---

## 一、产品概述

### 1.1 目标用户
- 小学生（主要使用者，打卡执行任务）
- 家长（管理活动、审核积分、代打卡、复盘点评）

### 1.2 核心价值
- 培养孩子时间管理能力
- 通过积分机制激励按时完成任务
- 家长可代打卡（解决孩子没手机的问题）
- 积分审核机制保证公平
- 长期目标 + 短期复盘形成闭环

### 1.3 技术栈
- Next.js 16 + TypeScript + Tailwind CSS 4 + shadcn/ui
- Prisma ORM + SQLite（v1.x）/ PostgreSQL（v2.0+）
- Zustand（状态管理）
- Playwright（端到端测试）

## 三、功能模块

### 3.1 日程管理

#### 3.1.1 四种周期
- **日度**（daily）：每天重复，需设置计划时间 + 截止时间
- **周度**（weekly）：每周指定星期几（1-7）执行
- **月度**（monthly）：每月指定日期（1-31）执行
- **临时**（once）：一次性活动，需指定具体日期（specificDate），只在该天出现

#### 3.1.2 活动字段
- 标题（必填）
- 描述（可选）
- 周期类型（日度/周度/月度/临时）
- 计划时间（HH:mm，可选）
- 截止时间（HH:mm，超过则不算按时，所有周期类型都可设，建议设置）
- 截止日期（endDate，可选，超过此日期活动自动停止，如学期末）
- 基础积分（完成即得）
- 按时奖励（截止时间前完成才得）
- 分配给孩子（支持多选：assignedToIds 数组）
- 具体日期（specificDate，仅临时类型必填）

#### 3.1.3 活动分配（多孩）
- **指定孩子**：活动可分配给一个或多个孩子（assignedToIds 逗号分隔）
- **公共活动**：assignedToIds 为空 = 所有孩子都需要打卡
- 公共活动每个孩子可独立打卡（各自的 ActivityLog）
- 非分配的孩子看不到该活动
- API 过滤：GET /api/activities?assignedToId=xxx 返回该孩子的活动 + 公共活动

#### 3.1.4 两种视图
- **列表视图**：按日度/周度/月度 Tab 分组，卡片式展示
- **时间网格视图**（默认）：纵向时间轴，活动条高度=时间跨度
  - 智能时间范围（根据当天活动自动收缩）
  - 重叠活动并排分栏
  - 当前时刻红色指示线
  - 家长可拖动条目上下调整时间（5 分钟对齐）
  - **日视图/周视图切换**：
    - 日视图：左右箭头换前一天/后一天，"回到今天"快捷按钮
    - 周视图：7 行紧凑俯览（周一到周日），每行显示该天活动清单
    - 周视图点击某天可切到日视图
  - **不区分周期类型**：网格显示选中天的所有活动（日度+周度+月度+临时按"是否在该天活跃"过滤）
  - **家长成员选择**：顶部孩子头像按钮，切换看谁的活动（默认第一个孩子）

#### 3.1.5 业务规则
- 周度活动必须指定 dayOfWeek（1-7）
- 月度活动必须指定 dayOfMonth（1-31）
- 临时活动必须指定 specificDate
- 非法 scheduleType 应拒绝
- 删除活动为软删除（active=false）
- 所有周期类型都可设截止时间（deadline）
- 所有周期类型都可设截止日期（endDate）
- 超过截止日期的活动自动停止出现

### 3.2 打卡与积分

#### 3.2.1 打卡流程
1. 孩子自己打卡 **或** 家长代打卡（记录 operatorId）
2. 打卡后状态 = `pending_verification`（待审核），**不立即发积分**
3. 家长审核通过 → 状态 = `completed`，发放基础积分 + 按时奖励
4. 家长审核拒绝 → 状态 = `rejected`，不发积分
5. 拒绝后可重新打卡

#### 3.2.2 按时奖励逻辑
- 打卡时当前时间 ≤ deadline → `onTime=true`，bonusAwarded = onTimeBonus
- 打卡时当前时间 > deadline → `onTime=false`，bonusAwarded = 0
- 审核通过时按 onTime 字段发放

#### 3.2.3 重复打卡限制
- 同一活动同一周期（日度=当天/周度=当周/月度=当月）只能打卡一次
- 状态为 `pending_verification` 或 `completed` 时拒绝重复打卡
- 状态为 `rejected` 或 `missed` 时允许重新打卡

#### 3.2.4 扣分机制
- 家长触发"扣分检查" → 找出所有"今日该出现但未打卡且超时"的活动
- 每个未完成活动扣除其基础积分（负数）
- **积分扣到 0 为止，不会变负数**
- 已打卡（待审核/已通过/已拒绝）的活动不会被扣分

#### 3.2.5 积分流水
- 每次积分变动都写流水（PointTransaction）
- 类型：earn（完成）/ bonus（按时奖励）/ penalty（扣分）/ redeem（兑换）/ adjust（手动调整）
- 手动调整积分（PATCH member.totalPoints）也必须写 adjust 流水
- 流水按时间倒序查询

### 3.3 批量审核
- 家长首页可见"待审核打卡"面板（始终显示，含空状态）
- 支持全选/单选
- "全部通过"按钮：无需勾选即可一键通过所有
- "拒绝选中"按钮：需先勾选
- 审核后面板实时刷新
- 代打卡记录显示"代打卡"标签

### 3.4 兑换系统

#### 3.4.1 奖励字段
- 标题、描述、图标、所需积分、创建者

#### 3.4.2 兑换流程
1. 孩子提交兑换申请（积分足够才能提交）
2. 提交时立即扣除积分
3. 家长审核：通过 → approved → 标记已兑现 fulfilled
4. 家长拒绝 → rejected → **退还积分**
5. 已 fulfilled 的兑换再次操作不应退分（幂等）

#### 3.4.3 业务规则
- 积分不足时兑换应失败
- 兑换记录包含状态：pending / approved / rejected / fulfilled
- 只能用"已审核积分"（totalPoints 仅含已审核的）

### 3.5 鼓励里程碑
- 达到累计积分阈值自动解锁称号
- 默认阈值：20/50/100（v1.x 有 5 个，固定 seed 有 3 个）
- 字段：threshold、title、message、icon
- 家长可自定义新增
- 按threshold 升序展示

### 3.6 规划功能

#### 3.6.1 个人长期目标
- 字段：标题 + 完成状态 + 截止日期 + 描述 + 归属成员
- 完成状态：未开始 / 进行中 / 已达成
- 每个成员独立目标（妈妈/爸爸/孩子各有）
- 按状态分组展示，支持快速切换
- 过期目标显示"已过期"标签（未达成且 deadline 已过）
- 支持成员过滤（全部/各成员）
- CRUD 完整：新建/编辑/删除
- 默认状态为 not_started
- 截止日期可清空（设为 null）
- 编辑时不可改归属成员

#### 3.6.2 复盘（数据统计）
- 周报/月报切换
- 4 个数字卡片：
  - 任务完成率 = 已完成 / 总任务数
  - 按时完成率 = 按时完成 / 已完成
  - 净增积分 = 获得 - 扣除 - 兑换
  - 未完成/扣分数
- recharts 趋势折线图：最近 4 个周期
  - 三条线：完成率、按时率、净积分
  - 周报 label 格式 "M/D"，月报 label 格式 "X月"
- 时间导航：可查看历史周期（offset 0=当前, 1=上一个...）
- 家长可查看任意成员的统计
- 月报 periodStart 必须是月初（1 号）
- 周报 periodStart 是周一

#### 3.6.3 复盘点评
- 点评周期（针对本周/本月整体表现）
- 所有成员都能写点评（含孩子自评）
- 周报点评 / 月报点评分离
- 历史点评列表按时间倒序
- 每条点评显示：作者、角色、周期、日期、内容
- 支持编辑内容（PATCH）
- 点评返回 author 关联数据

### 3.7 家庭管理

#### 3.7.1 成员主题色（v2.3.0 新增）
- **6 套预设主题**：活力橙（🟠）、甜美粉（🌸）、清新绿（🌿）、天空蓝（💙）、梦幻紫（🔮）、热情红（🍎）
- **影响范围**：切换成员后整个 App 配色跟着改变（primary/ring/chart-1/sidebar-primary 等 CSS 变量）
- **存储方式**：Member 表新增 `theme` 字段（key: orange/pink/green/blue/purple/red），兼容旧数据用 `color` 反查
- **配色选择入口**：
  - 家庭 Tab 每个成员卡片有"配色"按钮（Palette 图标）
  - 点击弹出 ThemePickerDialog，选择预设主题并保存
  - 成员对话框（添加/编辑）中也有主题色选择器
- **权限**：
  - 家长可改任意成员的主题
  - 孩子可改自己的主题（theme-only 更新，API 支持自更新）
- **兼容旧数据**：旧成员无 theme 字段时，用 `guessThemeByColor(color)` 反查最接近的主题
- **未登录时清除主题**：回到 CSS :root 默认值（橙色）
- 成员 CRUD：新增/编辑/删除
- 成员字段：姓名、角色、头像、主题色
- 非法 role 应拒绝
- 角色切换（前端 Zustand 持久化 currentMemberId）
- 积分流水查看（按成员）
